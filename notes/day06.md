	•	how to store chat history in st.session_state: We can use a list to store the messages, and append to it every time the user or bot sends a message. Each entry in the list will be a dictionary with the following keys: role (the author of the message), and content (the message content).
	•	how widgets/reruns affect state: In Streamlit, interacting with a widget (e.g., clicking, typing) updates its value in st.session_state and triggers a full script rerun from top to bottom, allowing the app to reflect changes based on the new input. Reruns can cause uncontrolled widgets to reset, making session state management essential to persist data.
	•	one caching idea to avoid recomputing retrieval every rerun: A highly effective method to avoid recomputing retrieval is implementing a Cache-Aside (Lazy Loading) pattern with content-addressed storage (like Redis or a local disk cache). Before running a search, the system generates a hash of the query parameters and checks if the result exists; if not, it computes it once and stores the result for future runs.
